# Devlog — 2026-03-20

## Session summary
Scaffolded the full Flask backend from scratch, based on the finalized schema and
conventions in CLAUDE.md. The frontend (React + Vite) was already in place; this
session focused entirely on the backend.

---

## What was done

### Entry point + app factory
- `run.py` — Flask entry point, calls `create_app()`, listens on port 5000
- `app/__init__.py` — app factory (`create_app`), wires up Flask-SQLAlchemy,
  Flask-Migrate, Flask-CORS, registers all blueprints under `/api/`

### Configuration
- `app/config.py` — `BaseConfig`, `DevelopmentConfig`, `ProductionConfig`
- All DB credentials and secrets read from env via `python-dotenv`; nothing hardcoded
- `.env.example` added as a safe-to-commit template

### Models (`app/models/`)
One file per domain entity, matching the finalized schema exactly:

| File | Models |
|---|---|
| `clinician.py` | `Clinician`, `ClinicianSchedule`, `ClinicianHMO`, `ClinicianInfo`, `ClinicianTimeslot` |
| `secretary.py` | `Secretary`, `SecretaryClinicianLink` (M2M junction) |
| `patient.py` | `Patient` |
| `appointment.py` | `Appointment` |

Design decisions preserved:
- `VARCHAR` everywhere (no ENUMs)
- Child tables for HMO and Info (not JSONB arrays)
- `ClinicianTimeslot` as a pre-generated slots table; `Appointment` references a slot
- `SecretaryClinicianLink` M2M junction, supports >1 secretary per clinician

### Routes (`app/routes/`)
One Blueprint per domain entity, all registered under `/api/`:

| Blueprint | Prefix | Notes |
|---|---|---|
| `clinician_routes` | `/api/clinicians` | CRUD + nested `/schedules`, `/hmos`, `/infos` |
| `secretary_routes` | `/api/secretaries` | CRUD + link/unlink clinician endpoints |
| `patient_routes` | `/api/patients` | Full CRUD |
| `timeslot_routes` | `/api/timeslots` | CRUD + query-param filters (`clinician_id`, `date`, `status`) |
| `appointment_routes` | `/api/appointments` | CRUD; create guards slot availability + marks slot `booked`; delete is soft-cancel (frees slot) |

Health check: `GET /api/health → { "status": "ok" }`
