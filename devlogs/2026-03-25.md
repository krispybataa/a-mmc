## Design rationale

### triageLogic.js as a domain-gated file
The triage routing logic is treated as a clinical decision, not a technical one.
Scaffolded with a mandatory warning block and placeholder values so the UI can be
built and evaluated independently of routing correctness. Domain expert sign-off
required before any real patient-facing deployment.

### Unauthenticated browsing allowed, gate only at booking
Patients can browse /find, /find/triage, /doctors, and /clinician/:id without an
account. Auth gate appears only at "Book Appointment", triggering the ?redirect= chain.
Avoids frustrating unfamiliar users (primary demographic: 60+) before they find
the right doctor.

### AppointmentDrawer consolidating actions
Replaces alert() with a readable summary of all appointment fields in one place,
consolidating Cancel and Request Reschedule into a single surface. Row-level shortcut
retained for users who know what they want without opening the drawer.

### Desktop-first degrading to mobile
Reflects deployment context (hospital kiosk + personal device) and primary demographic
(60+ users on varied devices). Touch targets minimum 44px, base font minimum 16px.
