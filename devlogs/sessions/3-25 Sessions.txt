✻ Churned for 2m 21s

● Fixed. The useParams() key is clinician_id (the route is /clinician/:clinician_id), so params.id from the bug
  report would have been wrong — the correct form is Number(clinician_id). All three mock clinicians (ids 1, 2, 3)
  will now resolve correctly; /clinician/99 still hits the not-found state since Number("99") won't match any entry.

● Root cause: App.jsx defines the route as /clinician/:id, so useParams() exposes { id: "1" } — not clinician_id.
  Destructuring clinician_id always gave undefined, making the find return nothing.

  Fix: const { id: clinician_id } = useParams() — destructures the id param and aliases it locally to clinician_id,
  so the rest of the component is unchanged. /clinician/1, /clinician/2, /clinician/3 all resolve now; /clinician/99
  still hits the not-found state.
