## Key design decisions made this session

| Decision | Detail |
|---|---|
| Token in memory, refresh in cookie | XSS protection — access token never touches localStorage |
| `configureApiAuth` module bridge | api.js can't use hooks; module-level refs synced via effect |
| Identical 401 for all auth failures | No user enumeration on login or staff login |
| Post-commit email calls | Failed send never affects appointment state |
| `commit=False` on `generate_slots` | Enables parent-transaction participation without behavior change at existing call sites |
| `window.location.href` for logout | AuthProvider outside BrowserRouter; hard nav flushes all React state |
| Stuck slots surfaced, not blocking | C/S informed of scheduling conflicts without preventing schedule save |
