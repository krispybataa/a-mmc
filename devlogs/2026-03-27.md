## Key design decisions made this session

| Decision | Detail |
|---|---|
| Smoke test before integration | Caught 3 backend bugs before any frontend work — right order |
| Inline cancel confirm over window.confirm() | Better UX, consistent across patient and C/S sides |
| Promise.allSettled() for schedule save | Per-row failure isolation — one bad row doesn't block others |
| Immediate add/remove for HMOs and infos | Discrete operations don't need a batch save button |
| Role-based clinician_id resolution | Secretary fetches linked clinician_id; clinician uses own id |
| availableSlots prop on SlotPicker | Backward compatible — RescheduleModal still generates from schedule |
| Refresh endpoint returns user identity | Fixed silent user=undefined bug on every page reload |
