USERS: 
	Clinician: 
	- title, firstname, lastname, middlename, suffix [robust name fields, essentially] 
	- department 
	- specialty/practice
	- Local Number [this is NOT a phone number] 
	- Room Number [ex. "Hall A Rm 230"] 
	- Schedule [From Mon-Sat and contains AM/PM, usually displayed in a table and ranges, refer to attached picture] 
	- Profile Picture 
	- Contact Details (phone, email (separate from login but allows it to be the same)) 
	- HMO Accreditaition (array of text displayed that is flexible) (can be empty) 
	- "More information" (same with HMO) (can be empty) 
	- login credentials (email,pwd) 

	Secretary: 
	(Mainly an assistive administrative role, has the same permissions as Clinician, and if a diagram is drawn, is attached to their respective clinician) 
	- title, firstname, lastname, suffix
	- Contact Details (phone, email (separate from login but allows it to be the same)) 
	- login credentials (email,pwd) 

	-> To elaborate on the secretary role, they basically manage the clincian's profile, so they can edit and update schedules, accept/reschedule appointments.

	Appointments:

	An appointment is booked within time frames (ex. 13:00-14:00) 
	- Clinic (This may be autofilled since a patient books an appointment within a doctor's profile, so the system should pull data wherever the patient is)

	- Appointment Type /*/
